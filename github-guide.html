Публикация сайта на GitHub Pages

GitHub Pages - это бесплатный хостинг для статических сайтов от GitHub. Вот пошаговая инструкция по публикации вашего сайта:

Шаг 1: Создание аккаунта на GitHub

Перейдите на GitHub и зарегистрируйте новый аккаунт, если у вас его еще нет.

Шаг 2: Создание нового репозитория
  1. Нажмите кнопку "+" в правом верхнем углу и выберите "New repository"
  2. Введите имя репозитория в формате: вашлогин.github.io (например, ivanov.github.io)
  3. Выберите "Public" (публичный репозиторий)
  4. Поставьте галочку "Add a README file"
  5. Нажмите "Create repository"
Шаг 3: Загрузка файлов вашего сайта через интерфейс GitHub
  1. В вашем репозитории нажмите "Add file" → "Upload files"
  2. Перетащите файлы вашего сайта (index.html, стили, изображения и т.д.)
  3. Нажмите "Commit changes"
Шаг 4: Настройка GitHub Pages
  1. В репозитории перейдите во вкладку "Settings"
  2. В меню слева выберите "Pages"
  3. В разделе "Source" выберите ветку "main" и папку "/ (root)"
  4. Нажмите "Save"

Через несколько минут ваш сайт будет доступен по адресу: https://вашлогин.github.io

Примечание: Главная страница вашего сайта должна называться index.html и находиться в корневой папке репозитория.
Шаг 5: Обновление сайта

Когда вы вносите изменения в файлы сайта, вам нужно загрузить их на GitHub:

Через интерфейс GitHub загрузите новые версии файлов

Шаг 6: Дополнительные настройки (опционально)

Вы можете добавить собственный домен к вашему GitHub Pages сайту:

  1. Купите домен у регистратора (например, reg.ru, nic.ru)
  2. В настройках DNS вашего домена добавьте 4 записи A с IP-адресами GitHub:
    • 185.199.108.153
    • 185.199.109.153
    • 185.199.110.153
    • 185.199.111.153
  3. В настройках GitHub Pages укажите ваш домен
  4. Создайте в корне репозитория файл CNAME с именем вашего домена

Поздравляем! Теперь ваш сайт доступен в интернете абсолютно бесплатно. Вы можете продолжать его развивать и обновлять по мере необходимости.
